## üìù Documentation Standards

### File Naming
- Use descriptive, lowercase names with hyphens
- Include dates for time-sensitive documents (YYYYMMDD format)
- Prefix with category when helpful (e.g., `api-`, `guide-`)

### Content Structure
- Start with purpose and scope
- Include table of contents for long documents
- Use consistent markdown formatting
- Update "Last Updated" dates

### Archive Policy
- Move outdated documents to `archive/`
- Maintain historical session logs for reference
- Keep documentation current and relevant
